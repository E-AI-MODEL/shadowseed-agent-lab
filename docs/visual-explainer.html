The invariant

Presence is not influence.

trace > 0 The seed is present or observed. It is a candidate absence, not a trusted fact.
weight = 0 The seed does not steer retrieval, answers, probes, warnings, tools, or decisions.
Plain-language version: the system may notice that something appears missing, but noticing is not permission to act on it.
The safe path

Input to evidence to gate to audit.

1

Input/context

A user message, document, ticket, note, or incident record starts the workflow.

2

Candidate absence

The system detects a small missing structure, such as an owner or deadline.

3

Weightless seed

The seed is recorded with trace, but starts with no steering power.

4

Evidence lookup

Lookup is driven by input/context, not by the seed steering retrieval.

5

Gate check

Verified evidence is evaluated by a gate-policy seam.

6

Audit replay

Every allowed decision must be replayable and tied to the same seed and gate.

Gate outcome

The fork that keeps agents safe.

Blocked path

No steer
  • No retrieval steering from a weightless seed.
  • No answer-text change.
  • No warning, probe, tool call, or downstream behavior.
  • Decision is still logged for audit.

Promoted path

Suggest
  • Seed has weight greater than zero.
  • Seed status is promoted.
  • Gate event is logged and matches the seed.
  • Only a bounded suggestion-only probe is allowed.
What the lab tests

Three guardrails for the test market.

🔎

Evidence is input-driven

RAG can provide evidence from the input context, but it does not assign weight or promote a seed.

🚧

Weightless means blocked

A seed with weight at zero is present only. It cannot change behavior.

🧾

Allowed means replayable

Audit replay checks that the decision seed, session seed, gate seed, and gate reference match.

20–40 sec
