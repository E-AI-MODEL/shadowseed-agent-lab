Repo 1: E-AI-MODEL/shadowseed

SSL is the research pipeline.

  • Defines what a shadow seed is and why one seed must equal one gap.
  • Separates trace as presence from weight as influence.
  • Owns lifecycle, Validation Gate, contradiction handling, probes, and evidence layers.
  • Measures progress through explicit layers instead of one blended score.
Repo 2: shadowseed-agent-lab

Agent lab is the application path.

  • Tests how an agent/RAG layer can use SSL without letting weightless seeds steer.
  • Provides input-driven evidence adapters and fixture-only gate seams.
  • Produces suggestion-only probes rather than external execution.
  • Feeds reusable safety and audit lessons back toward upstream SSL when generic.
What distinguishes SSL

It treats absence as a testable object, not as vague context.

1 gapA seed must contain one small, specific, testable structural absence.
tracePresence and recurrence are tracked separately from influence.
weightInfluence starts at 0.0 and can rise only through the Validation Gate.
Short version: SSL does not add “more context”. It records a precise missing relation, tests it, and only then may use it to navigate.
Golden path

How the safe application flow works.

1

Input/context

A user message, document, ticket, note, or incident record starts the workflow.

2

Candidate absence

The system detects one small missing structure, such as an owner, deadline, source, or constraint.

3

Weightless seed

The seed is recorded with trace. It has no steering power.

4

Evidence

Evidence lookup is driven by input/context, not by the seed itself.

5

Gate

The gate checks evidence and contradiction before promotion.

6

Audit

Allowed decisions must replay to the same seed and gate reference.

Gate outcome

The fork that keeps agent use safe.

Blocked path

No steer
  • No retrieval steering from a weightless seed.
  • No answer-text change.
  • No warning, probe, tool call, or downstream behavior.
  • The decision is still logged for audit.

Promoted path

Suggest
  • Seed has weight greater than zero.
  • Seed status is promoted.
  • Gate event is logged and matches the seed.
  • The lab allows bounded suggestion-only probes.

What does not exist yet

Make the open edges visible.

  • No production agent that executes external actions.
  • No claim that SSL is fully validated across all open-ended model tasks.
  • No replacement of the Dutch 4.5/4.6 SSOT documents by this English HTML page.
  • No permission for weightless seeds to steer retrieval, answers, warnings, tools, or decisions.
  • No claim that one demo or fixture-smoke equals scientific end validation.
90-second test-market script

How to present it.

0–20 sec

Start with the idea: models do not only produce answers; they can also notice precise structural absences.

20–40 sec

Explain atomic seeds: one seed is one gap. Trace records presence; weight records influence and starts at zero.

40–65 sec

Show the research pipeline: validation, evidence layers, contradiction handling, and payoff testing.

65–85 sec

Show the application pipeline: agent-lab wraps SSL into input-driven evidence, gate checks, suggestion-only probes, and audit replay.

85–90 sec

Close honestly: promising mechanism, active evidence pipeline, not a production claim. For exact wording, the Dutch SSOT remains leading.
